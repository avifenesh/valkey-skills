Key decisions:
- **`port 0` + `tls-port 6379`**: Disables plaintext; all traffic over TLS
- **`tls-replication yes` + `tls-cluster yes`**: Encrypts inter-node replication and gossip bus
- **`tls-auth-clients no`**: Server-only TLS (no mTLS for app clients) - simplifies client setup
- **`noeviction`**: Prevents silent data loss from search indexes
- **`valkey-bundle`** image has the module at `/opt/valkey/modules/valkeysearch.so`

## Deployment Guide

### Prerequisites

- Kubernetes 1.20+
- `kubectl` connected to a cluster
- `openssl` for TLS cert generation
- ~60 GB storage (6 pods x 10 GB PVC)

### Deploy

```bash
cd examples/kubernetes-cluster-deployment

# 1. Generate TLS certs and K8s Secret manifest
bash generate-tls-certs.sh valkey

# 2. Deploy everything (or use the orchestration script)
kubectl apply -f valkey-namespace.yaml
kubectl apply -f valkey-tls-secret.yaml
kubectl apply -f valkey-configmap.yaml
kubectl apply -f valkey-statefulset.yaml

# 3. Wait for all 6 pods
kubectl wait --for=condition=ready pod \
  -l app.kubernetes.io/name=valkey -n valkey --timeout=300s

# 4. Initialize cluster (auto-detects if already done)
kubectl apply -f valkey-cluster-init.yaml
kubectl wait --for=condition=complete job/valkey-cluster-init -n valkey --timeout=120s

# 5. Run search tests
kubectl apply -f valkey-test-job.yaml
kubectl logs -f job/valkey-search-test -n valkey
```

Or use the orchestration script: `bash deploy-cluster.sh valkey deploy`
