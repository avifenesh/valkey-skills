The implementation is complete. Here's what I built:

- **TaskQueue** — produces 100 tasks via `xadd` with high/medium/low priorities
- **Worker** — consumer group reader with:
  - Idempotent `XGROUP CREATE` with `MKSTREAM`
  - Blocking `XREADGROUP` (2s timeout for responsive shutdown)
  - `XACK` after processing
  - Dead letter: `XPENDING` range + `XCLAIM` for stale messages (>30s idle), moves to DLQ after 3 retries
- **3 concurrent workers** — each with its own `GlideClient` connection (required for blocking reads)
- **Dashboard** — every 5s showing stream length, pending counts per consumer, DLQ size, processed total
- **Graceful shutdown** — `SIGTERM`/`SIGINT` handlers stop workers, wait for in-flight work, print final dashboard, close connections
- **Auto-exit** — shuts down cleanly after all 100 tasks are processed

Test with:
```bash
docker compose up --build
```
