**Migration summary - `app.js` (ioredis → Valkey GLIDE):**

| Feature | Approach |
|---|---|
| **Connection** | `GlideClusterClient.createClient()` with 3 node addresses |
| **Cache with TTL** | `customCommand` for SET with EX/NX options; built-in `mget` for multi-get |
| **Pub/Sub** | Subscriber created via `createClient` with `pubsubSubscriptions` config (pattern mode); publish via `customCommand` |
| **Batch ops** | `ClusterBatch` (falls back to `ClusterTransaction`) with `customCommand` for HSET/HGETALL |
| **Streams** | `customCommand` for XGROUP CREATE, XADD, XREADGROUP, XACK, XINFO; handles both RESP2/RESP3 response formats |
| **Sorted sets** | Built-in `zadd` with `{element, score}[]`; `customCommand` for ZREVRANGE, ZREVRANK, ZRANGEBYSCORE with WITHSCORES |

Key design decisions:
- Uses `import * as Glide` for defensive imports with fallbacks (`ClusterBatch || ClusterTransaction`, `PubSubChannelModes` with hardcoded fallback)
- Handles both RESP2 (flat arrays) and RESP3 (maps/objects) formats for `customCommand` responses
- Pub/Sub uses creation-time subscription config with callback, plus a 10s safety timeout
- 500ms delay before publishing to let subscription propagate
