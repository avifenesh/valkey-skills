Implementation complete. Here's what was built:

**`queue.js`** — Full message queue system using `@valkey/valkey-glide`:

- **TaskQueue** — Producer that adds tasks via `xadd` with type, priority, payload, and timestamp fields
- **Worker** (x3) — Consumer group readers using `xreadgroup` with blocking reads (2s timeout), `xack` after processing, and simulated 50-200ms work
- **Dead letter handling** — `xautoclaim` reclaims messages idle >30s; retry count tracked in a Valkey hash; messages exceeding 3 retries are moved to `tasks:deadletter` stream
- **Dashboard** — Every 5s prints stream length, processed count, pending messages, DLQ count, and per-consumer stats via `xlen`/`xpending`
- **Graceful shutdown** — SIGTERM/SIGINT handlers stop workers, wait for in-flight work, print final dashboard, close all connections
- **4 GLIDE clients** — 1 for producer/dashboard, 1 per worker (dedicated for blocking reads as recommended)
- **Auto-exit** — Shuts down cleanly after all 100 tasks are processed

**`docker-compose.yml`** — Added `init: true` (tini for signal forwarding) and `exec` so `node` receives SIGTERM directly.

**Robust parsing** — Stream result parsers handle multiple GLIDE return formats (Record, GlideRecord `{key, value}[]`, array-of-tuples, flat arrays) so the code works across GLIDE versions and protocol modes.

Test with:
```bash
docker compose up --build
```
