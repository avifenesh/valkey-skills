The implementation is complete. Here's what was built:

**`TaskQueue`** — `addTask` calls `xadd` with typed fields; `addBatch` iterates sequentially.

**`Worker`** — Each worker runs its own blocking `xreadgroup` loop with `block: 2000` so it can check `this.running`. Before each read, it calls `claimStale()` which uses `xautoclaim` with a 30s idle threshold. Retry counts are tracked in a `tasks:retries` hash; once a message hits 3 retries it's moved to `tasks:deadletter` via `moveToDLQ`.

**3 dedicated clients** — Blocking `xreadgroup` calls hold the connection open, so each worker gets its own `GlideClient` instance. The producer/dashboard shares a fourth client.

**Dashboard** — Prints every 5s via `setInterval`: stream length (`xlen`), unacked count (`xpending` summary), DLQ depth, and per-worker processed counts.

**Graceful shutdown** — `SIGTERM`/`SIGINT` set `running = false` on all workers, waits for `Promise.allSettled`, prints final dashboard, closes all clients. Also auto-exits when all 100 tasks are processed and pending = 0.

**Key GLIDE details applied:**
- `xreadgroup(group, consumer, {streamKey: ">"}, {count, block})`
- Fields from stream come back as flat `[f1,v1,f2,v2]` arrays → `fieldsToObject()` converts them
- `xgroupCreate` with `{ mkStream: true }` to create stream atomically
- `hincrby` on a hash for atomic retry counting
- `xadd` accepts plain `{field: value}` objects
