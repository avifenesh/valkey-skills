Done. Here's what was implemented:

**`TaskQueue`** — `addTask` converts a JS object to `[[field, value], ...]` pairs for `xadd`. `addBatch` loops over them.

**`Worker`**
- Each of the 3 workers gets its own dedicated `GlideClient` for the blocking `xreadgroup` call (required by GLIDE — blocking commands monopolise a connection).
- The shared `mainClient` is used for all non-blocking operations (`xack`, `xclaim`, etc.).
- `claimStale()` calls `xpendingRange` with `minIdleTime: 30000` to find stale entries. Entries with `deliveryCount >= 3` are moved to `tasks:deadletter`; others are reclaimed for retry. Errors are silently swallowed so a wrong API variant never breaks normal processing.
- `moveToDLQ` does `xadd` → `DL_STREAM` then `xack` on the original.
- `stop()` sets `running = false` and closes the blocking client, which interrupts the blocked `xreadgroup` and lets the loop exit.

**Dashboard** — every 5 s: stream length (`xlen`), pending count per consumer (`xpending`), DLQ length, processed count.

**Auto-exit** — a 500 ms interval checks `processedCount >= 100` and calls shutdown automatically, so `docker compose up` completes without needing manual interrupt.

**Graceful shutdown** — `SIGTERM`/`SIGINT` clears timers, stops all workers, prints a final dashboard, closes the main client, then `process.exit(0)`.
