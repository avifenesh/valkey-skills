All files created. Here's a summary of the deliverables:

### Files

| File | Purpose |
|---|---|
| `kind-config.yaml` | Kind cluster with 1 control-plane + 3 workers (labeled with zones for anti-affinity) |
| `Dockerfile.valkey-search` | Multi-stage build adding valkey-search module to Bitnami valkey-cluster image |
| `helm-values.yaml` | Bitnami `valkey-cluster` chart values covering all requirements |
| `deploy.sh` | End-to-end deployment: prereqs check, kind cluster, TLS certs, image build, Helm install, ACL hardening |
| `test.sh` | 14 validation tests covering every requirement |

### Architecture decisions

- **Bitnami `valkey-cluster` Helm chart** handles StatefulSet, cluster init, and service creation
- **3 primaries + 3 replicas** via `cluster.nodes: 6`, `cluster.replicas: 1`
- **ACL users** defined in `commonConfiguration` — admin (full), app (data only, no `@admin`/`@dangerous`), monitor (INFO/CLUSTER INFO/CLIENT LIST/PING only)
- **Default user** disabled post-deployment via `ACL SETUSER default off` on each node; custom probes use the `admin` user so they survive the change
- **TLS** — self-signed CA + SAN cert generated by `deploy.sh`, stored in a K8s Secret
- **valkey-search** — built from source in a multi-stage Dockerfile; falls back to stock image if compilation fails
- **HA** — soft pod anti-affinity + PDB with `minAvailable: 2`
- **Persistence** — 2Gi PVCs per node, AOF enabled (`appendonly yes`, `appendfsync everysec`)
- **Monitoring** — Bitnami metrics exporter sidecar on port 9121 with Prometheus scrape annotations

### Usage

```bash
./deploy.sh   # creates kind cluster + deploys everything
./test.sh     # validates all 14 checks
```
