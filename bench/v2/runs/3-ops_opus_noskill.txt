All files created. Here's a summary of the deployment:

**File structure:**
```
kind-config.yaml          # Kind cluster: 1 control-plane + 3 workers
k8s/
  namespace.yaml          # valkey namespace
  configmap.yaml          # valkey.conf, users.acl, startup.sh
  statefulset.yaml        # 6-replica StatefulSet + Prometheus exporter sidecar
  service-headless.yaml   # Headless service for cluster discovery
  service-metrics.yaml    # Metrics service (port 9121) for Prometheus scraping
  pdb.yaml                # PodDisruptionBudget (maxUnavailable: 1)
deploy.sh                 # Full deployment script
test.sh                   # Validation tests
```

**What's covered per requirements:**

| Requirement | Implementation |
|---|---|
| 3 primary + 3 replica | 6-pod StatefulSet, `--cluster-replicas 1` at formation |
| ACL (admin/app/monitor, default off) | `users.acl` in ConfigMap; admin=full, app=all minus admin/dangerous, monitor=info/ping/cluster info/client list |
| TLS everywhere | Self-signed CA + server cert with SANs for all pod FQDNs; `tls-cluster yes`, `tls-replication yes` |
| valkey-search module | Startup script auto-loads if found; tests skip gracefully if absent |
| PDB | `maxUnavailable: 1` ensures at least 2 primaries always up |
| Anti-affinity | Preferred pod anti-affinity on `kubernetes.io/hostname` |
| Liveness/readiness probes | Exec-based `valkey-cli --tls ... ping` |
| Persistent storage | PVC per pod (1Gi), AOF enabled |
| Prometheus metrics | `oliver006/redis_exporter` sidecar on port 9121 with dedicated scrape service |

**deploy.sh** handles: kind cluster creation, image loading, TLS cert generation, manifest application, cluster formation (hostname-first with IP fallback), and verification.

**test.sh** validates: cluster health (state=ok, 3 primaries), read/write as app user, search index + vector KNN query (if module loaded), ACL enforcement (monitor can't write, default user disabled, app can't run CONFIG).
