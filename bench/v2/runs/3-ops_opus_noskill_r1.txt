All files created. Here's a summary of what was built:

## Deliverables

### `kind-config.yaml`
Kind cluster with 1 control-plane + 3 worker nodes, each labeled with a different `topology.kubernetes.io/zone` for anti-affinity spreading.

### `helm-values.yaml` (Bitnami `valkey-cluster` chart)
- **Topology**: 6 nodes (3 primary + 3 replica, `cluster.replicas: 1`)
- **ACL users**: `admin` (full access), `app` (+@all -@admin -@dangerous), `monitor` (INFO/CLUSTER INFO/CLIENT LIST only). Default user restricted to internal-only commands for chart compatibility (effectively disabled for data ops).
- **TLS**: Enabled via pre-generated self-signed certs in a K8s Secret
- **AOF persistence**: `appendonly yes`, `appendfsync everysec`
- **PDB**: `minAvailable: 2`
- **Anti-affinity**: `podAntiAffinityPreset: soft`
- **Custom probes**: Liveness/readiness/startup all authenticate as `admin` user over TLS
- **Metrics**: Prometheus exporter sidecar on port 9121 with scrape annotations
- **Module**: `loadmodule` directive ready (commented out — requires image with valkey-search .so)

### `deploy.sh`
1. Checks prerequisites (kind, kubectl, helm, openssl)
2. Creates kind cluster (idempotent)
3. Generates self-signed CA + server TLS certs with proper SANs
4. Creates namespace and TLS Secret
5. Adds Bitnami Helm repo
6. `helm upgrade --install` with `--wait --timeout 10m`
7. Waits for all pods Ready, prints summary

### `test.sh` (11 tests)
1. Cluster health (state OK, 6+ nodes)
2. App user read/write (SET/GET)
3. Vector search index + Lua-based vector insertion & KNN query (skips gracefully if module not loaded)
4. Monitor user denied SET/DEL (NOPERM)
5. Monitor user allowed INFO/CLUSTER INFO/CLIENT LIST
6. App user denied SHUTDOWN/CONFIG SET
7. Default user denied SET
8. PVCs exist (≥6)
9. PDB exists
10. Metrics service/annotation
11. TLS connectivity verified
