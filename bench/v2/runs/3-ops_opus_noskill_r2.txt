All files are created. Here's a summary of the deliverables:

### Files Created

| File | Purpose |
|------|---------|
| `kind-config.yaml` | Kind cluster with 1 control-plane + 3 workers (zone-labeled for anti-affinity) |
| `Dockerfile.valkey-search` | Multi-stage build: compiles valkey-search module, copies into valkey:8.0 image |
| `acl-users.conf` | ACL configuration: `admin` (full), `app` (read/write+FT.*), `monitor` (read-only), default disabled |
| `helm-values.yaml` | Bitnami `valkey-cluster` chart values covering all requirements |
| `pdb.yaml` | Standalone PodDisruptionBudget (minAvailable: 2) |
| `metrics-service.yaml` | Prometheus scraping Service + optional ServiceMonitor CRD |
| `deploy.sh` | End-to-end deployment: kind cluster → custom image build → namespace/ACL → Helm install → verify |
| `test.sh` | Comprehensive validation: cluster topology, TLS, persistence, PVCs, PDB, anti-affinity, metrics, app read/write, vector search (create index/insert/query/FT.INFO), ACL enforcement |

### Key Design Decisions

- **Bitnami `valkey-cluster` Helm chart** — the most mature operator-style tool for Valkey on Kubernetes; manages StatefulSets, services, TLS auto-generation, and cluster initialization
- **Custom Docker image** with valkey-search module built from source; `deploy.sh` falls back to stock `valkey/valkey:8.0` if the module build fails
- **TLS**: `tls.autoGenerated: true` lets the chart's init containers create self-signed certs
- **ACL**: Mounted as a ConfigMap into `/opt/bitnami/valkey/etc/users.acl` via `extraVolumes`/`extraVolumeMounts`
- **Anti-affinity**: `podAntiAffinityPreset: soft` (kind has limited nodes) + `topologySpreadConstraints`
- **Metrics**: Bitnami exporter sidecar on port 9121 with `prometheus.io/scrape` annotations

### Usage

```bash
./deploy.sh   # Creates kind cluster, builds image, deploys everything
./test.sh     # Validates the deployment end-to-end
```
