Here's a summary of all deliverables:

### Files created

| File | Purpose |
|---|---|
| `kind-config.yaml` | Kind cluster with 1 control-plane + 3 worker nodes (for anti-affinity) |
| `valkey-values.yaml` | Bitnami `valkey-cluster` Helm chart values |
| `Dockerfile.valkey-search` | Custom image extending Bitnami's with valkey-search module |
| `deploy.sh` | End-to-end deployment script |
| `test.sh` | Comprehensive validation script |

### What `deploy.sh` does
1. Checks prerequisites (kind, kubectl, helm, openssl, docker)
2. Creates a 4-node kind cluster (1 control-plane + 3 workers)
3. **Best-effort** builds a custom Docker image with the valkey-search module and loads it into kind
4. Generates self-signed TLS certificates (CA + server cert with proper SANs)
5. Creates the `valkey` namespace and TLS secret
6. Installs `bitnami/valkey-cluster` Helm chart with all configuration
7. Applies a Prometheus metrics scraping service
8. Waits for all 6 pods to be ready and verifies cluster health

### What `test.sh` validates
- **Cluster topology**: 6 nodes, 3 primaries, cluster state ok, all 16384 slots assigned
- **TLS**: Verified via TLS-only PING
- **AOF persistence**: Confirmed via CONFIG GET
- **PDB**: At least one PodDisruptionBudget exists
- **Anti-affinity**: Rules present on the StatefulSet
- **Liveness/readiness probes**: Configured on containers
- **Metrics**: Service and exporter sidecar present
- **ACL `app` user**: SET, GET, LPUSH/LLEN work; CONFIG GET blocked (NOPERM)
- **ACL `monitor` user**: INFO, CLUSTER INFO, CLIENT LIST allowed; SET blocked (NOPERM)
- **ACL `admin` user**: Full access (ACL LIST, CONFIG GET)
- **Vector search** (if module loaded): Creates index, inserts docs with embeddings, runs KNN query via Python test pod
- **Storage**: Verifies 6 PVCs exist

### Key design decisions
- **Bitnami Helm chart** chosen over raw operator — more mature, better documented, handles StatefulSet/cluster init automatically
- **TLS certs** generated by the deploy script (not cert-manager) for portability on kind
- **Default user** kept with chart-managed password for cluster bus auth; custom ACL users layered on top
- **`podAntiAffinityPreset: soft`** since kind only has 3 workers (use `hard` in production)
- **`maxUnavailable: 1`** in PDB guarantees at least 2 primaries survive voluntary disruptions
