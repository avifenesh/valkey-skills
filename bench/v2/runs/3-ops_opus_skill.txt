All files created. Here's a summary of the deliverables:

### Files

| File | Purpose |
|------|---------|
| `kind-config.yaml` | Kind cluster with 1 control plane + 3 workers (for anti-affinity) |
| `values.yaml` | Bitnami `valkey-cluster` Helm chart values |
| `Dockerfile.valkey-modules` | Custom image with valkey-search module (multi-stage build) |
| `deploy.sh` | Creates kind cluster, deploys everything, restricts default user |
| `test.sh` | Validates all requirements (cluster health, TLS, ACL, persistence, metrics, PDB, anti-affinity, vector search) |

### Architecture

- **Chart**: `bitnami/valkey-cluster` (operator-managed StatefulSet)
- **Topology**: 6 nodes total (3 primaries, 1 replica each)
- **TLS**: Auto-generated self-signed certs (`tls.autoGenerated: true`)
- **ACL**: 3 named users (`admin`, `app`, `monitor`) with appropriate permissions; default user restricted post-deploy to only health-check commands (ping/cluster/auth/info)
- **Custom probes**: Authenticate as `admin` user so they survive default-user restriction
- **Persistence**: 1Gi PVCs with AOF (`appendonly yes`, `appendfsync everysec`)
- **Metrics**: Prometheus exporter sidecar on port 9121
- **PDB**: `minAvailable: 4` (guarantees >= 2 primaries with failover)
- **Anti-affinity**: `soft` preset (kind has limited nodes; use `hard` in production)

### Usage

```bash
# Standard deploy (without valkey-search module)
./deploy.sh

# Deploy with valkey-search module (builds custom image)
./deploy.sh --with-modules

# Run validation tests
./test.sh
```
