All files are created and consistent. Here's a summary of the deliverables:

**Files created:**

| File | Purpose |
|---|---|
| `kind-config.yaml` | Kind cluster with 3 workers for anti-affinity |
| `namespace.yaml` | `valkey` namespace |
| `Dockerfile.valkey-search` | Multi-stage build adding valkey-search module to Bitnami image |
| `helm-values.yaml` | Bitnami `valkey-cluster` Helm chart values (all config) |
| `servicemonitor.yaml` | Prometheus ServiceMonitor CRD (applied if Operator exists) |
| `deploy.sh` | Full deployment: kind cluster, TLS certs, image build, Helm install, post-init hardening |
| `test.sh` | 9-section validation: cluster health, data ops, vector search, ACL, TLS, AOF, PVCs, PDB, metrics |

**Architecture decisions:**
- **Bitnami Helm chart** chosen over hand-crafted StatefulSets (no official Valkey K8s Operator exists yet)
- **Default user** is restricted via ACL (`resetkeys`, `-@write -@read -@scripting -@pubsub`) during init, then fully disabled post-formation via `ACL SETUSER default off`
- **Custom probes** use the `admin` user instead of default, so probes survive default-user disabling
- **Soft anti-affinity** because 6 pods on 3 workers requires co-location
- **PDB minAvailable=4** (out of 6) ensures at least 2 primaries survive voluntary disruptions
- **Search module**: custom Docker image built from source; deploy.sh falls back to standard image if build fails, and test.sh skips vector search tests gracefully
