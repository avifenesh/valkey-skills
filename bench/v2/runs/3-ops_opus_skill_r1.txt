### Architecture decisions

- **Bitnami `valkey-cluster` Helm chart** handles cluster mode, PDB, anti-affinity, TLS, metrics exporter, persistence, and probes natively via chart values.
- **ACL**: Default user is `on` with `resetkeys -@all` (no key access, no data commands) — effectively disabled for clients while preserving cluster-bus auth and health probes. Three named users (`admin`, `app`, `monitor`) with the specified permissions.
- **TLS**: `tls.autoGenerated: true` — the chart generates self-signed CA + server certs automatically.
- **valkey-search**: An init container copies modules from the official `valkey/valkey:8.1` image into a shared emptyDir volume. After pods start, `deploy.sh` attempts `MODULE LOAD` on each node. The test gracefully skips search tests if the module isn't available.
- **Vector search test**: Uses Lua `EVAL` to construct binary float32 vectors inside Valkey, avoiding bash null-byte limitations.
- **Metrics**: Prometheus exporter sidecar with `prometheus.io/scrape` annotations on the service.
