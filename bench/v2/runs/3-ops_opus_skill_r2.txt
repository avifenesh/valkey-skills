**Architecture decisions:**

- **Bitnami `valkey-cluster` Helm chart** (via OCI) — the most mature ecosystem tool for Valkey cluster mode on Kubernetes
- **ACL users** defined inline in `commonConfiguration` (not `aclfile`) — works without custom init containers
- **Default user disabled** post-deployment via `ACL SETUSER default off` on each node (the chart needs it active during cluster formation)
- **TLS** — self-signed CA + server cert with SANs for all intra-cluster DNS names
- **PDB** `maxUnavailable: 1` — guarantees at least 2/3 primaries survive voluntary disruptions
- **valkey-search** — optional via `BUILD_SEARCH_IMAGE=true` environment variable at deploy time

**Usage:**
```bash
# Standard deployment (no search module)
./deploy.sh

# With valkey-search module
BUILD_SEARCH_IMAGE=true ./deploy.sh

# Validate
./test.sh
```
