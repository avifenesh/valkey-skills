## Deliverables

**Operator choice**: [Hyperspike/valkey-operator](https://github.com/hyperspike/valkey-operator) — the most mature community operator (v0.0.61). The official `valkey-io/valkey-operator` is explicitly pre-release with no published images.

**Image**: `valkey/valkey-bundle` — official multi-module image that includes valkey-search 1.2.0 (requires Valkey 9.0.1+, which valkey-bundle provides). The Hyperspike default image is 8.1.4, so this override is necessary.

### Key design decisions in `k8s/02-valkey-cluster.yaml`

- `nodes: 6` → 3 primary + 3 replica in cluster mode
- `tls: true` → operator provisions certs via cert-manager
- `prometheus: true` → redis_exporter v1.78.0 sidecar on port 9121
- `disableDefaultUser: true` → default user disabled as required
- ACL: `admin ~* &* +@all`, `app` data commands only (no `@admin`/`@dangerous`), `monitor` read-only (`nocommands +INFO +CLUSTER|INFO ...`)
- `config.appendonly: yes` + `appendfsync: everysec` → AOF enabled
- `config.loadmodule: /usr/local/lib/valkey/modules/valkeysearch.so` → valkey-search loaded at startup
