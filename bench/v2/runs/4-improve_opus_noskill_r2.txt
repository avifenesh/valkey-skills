| # | Scenario | Verdict | Key Issue |

| 1 | Session Management | Problematic | `KEYS` blocks server; use `EXPIRE`/`EXPIREAT` instead of cron cleanup |
| 2 | Product Catalog Cache | Problematic | `DEL` in a loop blocks; use `UNLINK` with batching and pipelining |

| 6 | Real-time Inventory | Problematic | `KEYS` on every page load; use a hash per SKU + Lua sum |
| 7 | Job Queue | Problematic | LIST queues lack acknowledgment; use Streams with consumer groups |
| 8 | Conditional Updates | Problematic | WATCH retries under contention; use Lua compare-and-set |
| 9 | Full-text Search | Problematic | Loading all keys for string matching; build inverted index with sets |
| 10 | Pub/Sub Notifications | Problematic | Pub/Sub is fire-and-forget; use Streams for reliable delivery |
| 11 | Slow Commands | Partially problematic | Threshold too high, buffer too small, no `CLIENT SETNAME` |
| 12 | Leaderboard TTL | Problematic (misconception) | `EXPIRE` works on any key type including sorted sets |
