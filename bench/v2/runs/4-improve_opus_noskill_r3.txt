Done. ANSWERS.md has been written with assessments for all 12 scenarios. Key findings:

1. **Session Management** - Replace `KEYS` + cron with `SET ... EX` and use `EXPIRE` for extensions
2. **Catalog Cache** - Use `UNLINK` (async) instead of `DEL` in a loop; batch with pipelines; use hash tags for cluster slot co-location
3. **Hash Field TTL** - Valkey 9.0.3 supports `HEXPIRE` for per-field TTL; no need to split keys
4. **Rate Limiting** - Fix operation order (remove old before add), use Lua for atomicity, use smaller members
5. **Distributed Lock** - Atomic release via Lua script (check-and-delete) to fix the race condition
6. **Inventory** - Replace `KEYS` per page load with a hash-per-SKU (`HGETALL inventory:{sku}`)
7. **Job Queue** - Migrate from LIST to Valkey Streams with consumer groups (`XREADGROUP`/`XACK`)
8. **Conditional Updates** - Replace WATCH/MULTI/EXEC with a Lua compare-and-set script
9. **Full-text Search** - Use a proper search engine or build inverted indexes with sets (`SADD`/`SINTER`)
10. **Pub/Sub** - Replace with Streams for durable, replayable messaging with acknowledgment
11. **Monitoring** - Set `CLIENT SETNAME`, lower slowlog threshold, supplement with `LATENCY` subsystem
12. **Leaderboard TTL** - `EXPIRE` works on any key type including sorted sets; just set it directly
